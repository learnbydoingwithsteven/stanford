# Stanford CS Undergraduate Core Courses

**Required Foundation for CS Bachelor's Degree**

---

## 📋 Core Course Requirements

All CS majors must complete these 11 core courses that form the foundation of computer science education.

---

## 🎯 Course List

### Programming Foundation (CS106 Series)

#### **CS106A - Programming Methodology**
- **Units**: 3-5
- **Prerequisites**: None
- **Quarter**: Autumn, Winter, Spring
- **Language**: Python (previously Java)
- **Topics**:
  - Variables, control flow, functions
  - Data structures (lists, dictionaries)
  - Object-oriented programming basics
  - Problem decomposition
  - Software engineering principles
- **Workload**: 10-15 hours/week
- **Textbook**: *The Art and Science of Java* (Roberts)
- **Projects**: 8 assignments + final project
- **Learning Outcomes**:
  - Write well-structured programs
  - Understand algorithmic thinking
  - Debug and test code effectively
  - Apply programming to real-world problems

#### **CS106B - Programming Abstractions**
- **Units**: 3-5
- **Prerequisites**: CS106A or equivalent
- **Quarter**: Autumn, Winter, Spring
- **Language**: C++
- **Topics**:
  - Recursion and recursive problem solving
  - Data structures (stacks, queues, trees, graphs, hash tables)
  - Algorithm analysis and complexity
  - Pointers and memory management
  - Abstract data types
  - C++ Standard Template Library (STL)
- **Workload**: 15-20 hours/week
- **Textbook**: *Programming Abstractions in C++* (Roberts)
- **Projects**: 7 assignments + final project
- **Learning Outcomes**:
  - Master recursive thinking
  - Implement complex data structures
  - Analyze algorithm efficiency
  - Write efficient C++ code

#### **CS106X - Programming Abstractions (Accelerated)**
- **Units**: 3-5
- **Prerequisites**: Strong programming background
- **Quarter**: Autumn
- **Note**: Accelerated version of CS106B
- **Topics**: Same as CS106B but faster pace + additional advanced topics
- **Workload**: 20-25 hours/week

### Systems Foundation

#### **CS107 - Computer Organization & Systems**
- **Units**: 3-5
- **Prerequisites**: CS106B/X
- **Quarter**: Autumn, Winter, Spring
- **Language**: C and Assembly
- **Topics**:
  - C programming and memory management
  - Pointers, arrays, and structures
  - Assembly language (x86-64)
  - Computer architecture basics
  - Memory hierarchy and caching
  - Compilation and linking
  - Debugging with GDB
- **Workload**: 15-20 hours/week
- **Textbook**: *Computer Systems: A Programmer's Perspective* (Bryant & O'Hallaron)
- **Projects**: 7 assignments (heap allocator, text editor, etc.)
- **Learning Outcomes**:
  - Understand how programs execute on hardware
  - Master C programming and memory management
  - Debug low-level system issues
  - Optimize code for performance

#### **CS110 - Principles of Computer Systems**
- **Units**: 3-5
- **Prerequisites**: CS107
- **Quarter**: Winter, Spring
- **Language**: C/C++
- **Topics**:
  - Processes and threads
  - Concurrency and synchronization
  - Networking and sockets
  - File systems
  - Virtual memory
  - System calls and kernel interaction
  - Distributed systems basics
- **Workload**: 20-25 hours/week
- **Textbook**: Course reader + *Operating Systems: Three Easy Pieces*
- **Projects**: 8 assignments (shell, HTTP proxy, MapReduce, etc.)
- **Learning Outcomes**:
  - Build concurrent and networked systems
  - Understand operating system internals
  - Write thread-safe code
  - Design distributed applications

---

### Mathematical Foundation

#### **CS103 - Mathematical Foundations of Computing**
- **Units**: 3-5
- **Prerequisites**: CS106B/X (can be concurrent)
- **Quarter**: Autumn, Winter, Spring
- **Topics**:
  - Propositional and predicate logic
  - Proof techniques (induction, contradiction, etc.)
  - Set theory and functions
  - Relations and equivalence classes
  - Graphs and trees
  - Regular languages and finite automata
  - Context-free grammars
  - Computability theory basics
- **Workload**: 10-15 hours/week
- **Textbook**: Course reader
- **Assignments**: 8 problem sets
- **Learning Outcomes**:
  - Write rigorous mathematical proofs
  - Understand theoretical CS foundations
  - Analyze computational problems formally
  - Apply logic to computer science

#### **CS109 - Introduction to Probability for Computer Scientists**
- **Units**: 3-5
- **Prerequisites**: CS103, Calculus (MATH 51 or equivalent)
- **Quarter**: Autumn, Winter, Spring
- **Topics**:
  - Probability axioms and counting
  - Random variables and distributions
  - Expectation and variance
  - Joint distributions and independence
  - Conditional probability and Bayes' theorem
  - Central limit theorem
  - Markov chains
  - Applications to CS (algorithms, ML, etc.)
- **Workload**: 10-15 hours/week
- **Textbook**: Course reader
- **Assignments**: 8 problem sets + programming assignments
- **Learning Outcomes**:
  - Apply probability to CS problems
  - Analyze randomized algorithms
  - Understand statistical inference
  - Model uncertainty in systems

---

### Algorithms & Theory

#### **CS161 - Design and Analysis of Algorithms**
- **Units**: 3-5
- **Prerequisites**: CS103, CS109, CS106B/X
- **Quarter**: Autumn, Winter, Spring
- **Topics**:
  - Algorithm design paradigms:
    - Divide and conquer
    - Greedy algorithms
    - Dynamic programming
    - Graph algorithms
  - Complexity analysis (Big-O, Θ, Ω)
  - Sorting and searching
  - Shortest paths (Dijkstra, Bellman-Ford)
  - Minimum spanning trees (Kruskal, Prim)
  - Network flow
  - NP-completeness introduction
- **Workload**: 15-20 hours/week
- **Textbook**: *Algorithm Design* (Kleinberg & Tardos)
- **Assignments**: 8 problem sets + programming assignments
- **Learning Outcomes**:
  - Design efficient algorithms
  - Analyze time and space complexity
  - Prove algorithm correctness
  - Recognize problem patterns

---

### Additional Core Requirements

#### **CS111 - Operating Systems Principles**
- **Units**: 3-5
- **Prerequisites**: CS110
- **Quarter**: Spring
- **Topics**:
  - Operating system design principles
  - Process scheduling
  - Memory management
  - File systems
  - I/O systems
  - Security and protection
  - Virtualization
  - Case studies (Linux, Windows, etc.)
- **Workload**: 15-20 hours/week
- **Textbook**: *Operating Systems: Three Easy Pieces* (Arpaci-Dusseau)
- **Projects**: 4-5 major projects
- **Learning Outcomes**:
  - Understand OS architecture
  - Implement OS components
  - Optimize system performance
  - Design secure systems

## 📊 Core Curriculum Overview

### Course Sequence
```
Freshman Year:
├── CS106A (Programming Methodology)
└── CS106B/X (Programming Abstractions)

Sophomore Year:
├── CS103 (Mathematical Foundations)
├── CS107 (Computer Organization)
├── CS109 (Probability)
└── CS110 (Computer Systems)

Junior Year:
├── CS161 (Algorithms)
└── CS111 (Operating Systems)
```

### Prerequisites Chain
```
CS106A → CS106B → CS107 → CS110 → CS111
                    ↓
                  CS103 → CS161
                    ↓
                  CS109 ↗
```

- Debug complex programs

- Work with large codebases

### Systems Knowledge
- Understand computer architecture and organization
- Build concurrent and distributed systems
- Optimize code for performance
- Debug system-level issues

### Theoretical Foundation
- Analyze algorithm complexity
- Design efficient algorithms
- Write mathematical proofs
- Apply probability and statistics

### Problem Solving
- Decompose complex problems
- Choose appropriate data structures
- Apply design patterns
- Think algorithmically
